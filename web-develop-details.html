3. HTML5 Element "Text"

Text is a fundamental component of any webpage, and HTML5 provides a variety of tags to format and present text in different ways. Understanding how to use these tags correctly ensures that your content is accessible and well-structured.

  • Headings (< h1> to < h6>): Headings range from < h1> (the highest level) to < h6> (the lowest level). They structure the content and indicate the hierarchy of information.

  • Strong Importance (< strong>): The < strong> tag indicates that the enclosed text is of strong importance, usually rendering it in bold.
  • Quotes (< blockquote>): Used for block-level quotations, usually displayed with indentation.

5. How to Add Links in HTML Code

Links are a fundamental feature of HTML, allowing users to navigate between pages, sections of a page, or external resources. The < a> tag is used to create hyperlinks, with the href attribute specifying the destination URL. Links can be styled, opened in new tabs, and can even link to email addresses or phone numbers.

  • Basic Hyperlinks: The basic syntax for a link involves the < a> tag and the href attribute, which defines the link's destination.

    • Example:

    
< a href="https://www.example.com">Visit Example.com< /a>
  • Linking to Different Destinations:
      Internal Links: Link to another section within the same page using an ID.
      External Links: Link to an external website
      Email Links: Use mailto: to create a link that opens the user's email client.

    • Example:

    

< a href="#section2">Go to Section 2< /a>


< a href="https://www.google.com">Search on Google< /a>


< a href="mailto:someone@example.com">Send Email< /a>
  • Opening Links in a New Tab: Use the target="_blank" attribute to open the link in a new tab or window.

    • Example:

    
< a href="https://www.example.com" target="_blank">Open Example in New Tab< /a>

  • Anchors for Page Sections: Anchors can link to specific sections of a page by setting an id attribute on the target element.

    • Example:

    

< h2 id="section2">Section 2< /h2>


< a href="#section2">Go to Section 2< /a>

6. How to Insert Images in HTML

Inserting images into your HTML documents is a fundamental skill for creating visually appealing web pages. Images enhance the user experience and can convey information more effectively than text alone. In HTML, the < img> tag is used to embed images.

  • The < img> Tag:
      -> The < img> tag is a self-closing tag that does not require a closing tag.
        -> src (source): Specifies the path to the image file.
        -> alt (alternative text): Provides alternative text for the image if it cannot be displayed.
        -> width and height: Define the dimensions of the image.
  • Image File Formats:
      -> JPEG/JPG: Good for photographs and images with many colors
      -> PNG: Supports transparency and is good for images with fewer colors or text.
      -> GIF: Used for simple graphics and animations.
  • Using Local vs. Online Images:
      -> Local Images: Stored in the same directory or a subdirectory of your HTML file.
      -> Online Images: Accessed via a URL.
  • Responsive Images:
      -> Use CSS or HTML attributes like max-width: 100%; to ensure images scale properly on different screen sizes.

    • Example: Inserting an Image 

    
    < !DOCTYPE html>
    < html lang="en">
        < head>
            < meta charset="UTF-8">
            < meta name="viewport" content="width=device-width, initial-scale=1.0">
            < title>Inserting Images Example< /title>
        < /head>
        < body>

            < h1>My Favorite Place< /h1>
                < p>Here is a picture of my favorite place:< /p>

                < img src="beach.jpg" alt="A beautiful beach during sunset" width="600">

        < /body>
    < /html>

Explanation:

    src="beach.jpg": This specifies the location of the image file. The image "beach.jpg" should be in the same directory as the HTML file.
    alt="A beautiful beach during sunset": The alt attribute provides a description of the image, which is useful for screen readers and if the image fails to load.
    width="600": This sets the width of the image to 600 pixels. The height will automatically adjust to maintain the aspect ratio.

9.HTML Forms and Inputs

HTML forms are essential for collecting user input on web pages. They can be used for a variety of purposes, such as registering users, collecting feedback, or submitting orders. HTML5 introduced new input types and attributes that make forms more powerful and user-friendly.

  • Form Structure:
      -> The < form> tag is used to create an HTML form. Inside it, various input elements are used to capture different types of user data.

    Example

< label for="name">Name:< /label>
< input type="text" id="name" name="name">
< input type="submit" value="Submit">
< /form>
  • Common Input Types:
      -> Text Input (< input type="text">): For single-line text input.
      -> Password Input (< input type="password">): Masks the input with dots or asterisks.
      -> Email Input (< input type="email">): Validates that the input is in the format of an email address.
      -> Number Input (< input type="number">): Allows only numeric input.
      -> Date Input (< input type="date">): Provides a date picker.
      -> Submit Button (< input type="submit">): Submits the form.
  • Form Attributes:
      -> action: Defines where to send the form data.
      -> method: Specifies the HTTP method (GET or POST) used when submitting the form.
    -> placeholder: Provides a hint inside the input field about what type of input is expected.
        -> required: Makes a field mandatory for form submission.
  • Form Validation:
      -> HTML5 provides built-in form validation for input types like email, number, and url. Custom validation messages can be provided using the pattern attribute.

    • Example: Simple HTML Form

      -> Text Input: Collects a user's name.

      -> Submit Button: Sends the form data to the server.

JavaScript classes and factory functions are used for creating objects and managing complex data structures. This tutorial will explain how to define and use classes, and compare them with factory functions.

Example: JavaScript Classes and Factory Functions

We’ll define a Person class and a factory function to create person objects.

            
    // JavaScript Class
    class Person {
        constructor(name, age) {
            this.name = name;
            this.age = age;
        }

        greet() {
            console.log(`Hello, my name is ${this.name}.`);
        }
    }

    let person1 = new Person('Alice', 25);
    person1.greet(); // Outputs: Hello, my name is Alice.

    // Factory Function
    function createPerson(name, age) {
        return {
            name: name,
            age: age,
            greet: function() {
                console.log(`Hello, my name is ${this.name}.`);
            }
        };
    }

    let person2 = createPerson('Bob', 30);
    person2.greet(); // Outputs: Hello, my name is Bob.

Explanation: Classes vs. Factory Functions

Classes: Use the class keyword to define a class with a constructor and methods. Provides a blueprint for creating multiple objects with the same structure.

Factory Functions: Functions that return objects with specific properties and methods. Useful for creating objects without using the class syntax.

Summary: Choosing Between Classes and Factory Functions

Both classes and factory functions are useful for object creation and management. Classes offer a more formal approach with inheritance and method definitions, while factory functions provide a flexible, functional approach to object creation.

19. JSON Explained with JavaScript

Overview: Working with JSON

JSON (JavaScript Object Notation) is a lightweight data-interchange format that is easy to read and write for humans and machines. This tutorial will explain how to parse JSON data and convert JavaScript objects to JSON.

Example: JSON Parsing and Stringifying

We’ll demonstrate how to convert a JavaScript object to JSON and parse JSON back into a JavaScript object.

            
    // JavaScript Object
    let user = {
        name: "Jane",
        age: 28,
        email: "jane.doe@example.com"
    };

    // Convert JavaScript Object to JSON
    let jsonString = JSON.stringify(user);
    console.log(jsonString); // Outputs: {"name":"Jane","age":28,"email":"jane.doe@example.com"}

    // Convert JSON to JavaScript Object
    let jsonObject = JSON.parse(jsonString);
    console.log(jsonObject.name); // Outputs: Jane

Explanation: JSON Methods

JSON.stringify(): Converts a JavaScript object into a JSON string.

JSON.parse(): Converts a JSON string back into a JavaScript object.

Summary: Handling JSON Data

JSON is essential for data interchange in web applications. Mastering JSON operations allows you to easily convert between JavaScript objects and JSON, facilitating communication with web APIs and data storage.

20. JavaScript Error Handling - Try Catch Finally

Overview: Managing Errors

Error handling in JavaScript allows you to manage and respond to runtime errors gracefully. This tutorial will cover the try...catch...finally statement for handling exceptions and ensuring that code executes properly even when errors occur.

Example: Using Try Catch Finally

We’ll demonstrate how to use try...catch...finally to handle errors and ensure that clean-up code executes regardless of whether an error occurs.

            
    try {
        let result = riskyOperation();
        console.log(result);
    } catch (error) {
        console.error("An error occurred: ", error.message);
    } finally {
        console.log("Cleanup code runs regardless of error.");
    }

    function riskyOperation() {
        throw new Error("Something went wrong!");
    }

Explanation: Error Handling Structure

try Block: Contains code that might throw an error.

catch Block: Executes if an error is thrown in the try block. Handles the error and allows you to respond appropriately.

finally Block: Executes after the try and catch blocks, regardless of whether an error occurred. Used for clean-up code.

Summary: Effective Error Management

Proper error handling ensures that your application can handle unexpected situations gracefully. By using try...catch...finally, you can manage errors effectively and maintain application stability.

Overview: Handling Events

Event listeners allow you to execute code in response to user interactions or other events. This tutorial will cover how to add, remove, and manage event listeners in JavaScript.

Example: Adding Event Listeners

We’ll demonstrate how to use event listeners to handle user interactions such as clicks and form submissions.

Code Example: Event Listeners

            
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Event Listeners</title>
</head>
<body>
    <button id="clickMe">Click Me!</button>
    <form id="myForm">
        <input type="text" id="textInput" placeholder="Type something...">
        <button type="submit">Submit</button>
    </form>
    <p id="message"></p>

    <script>
        const button = document.getElementById('clickMe');
        const form = document.getElementById('myForm');
        const message = document.getElementById('message');

        // Adding a click event listener
        button.addEventListener('click', () => {
            message.textContent = "Button was clicked!";
        });

        // Adding a submit event listener to the form
        form.addEventListener('submit', (event) => {
            event.preventDefault(); // Prevent form submission

        });
    </script>
</body>
</html>

Explanation: Event Handling

Adding Event Listeners: Use addEventListener() to attach event handlers to elements.

Event Types: Common events include click, submit, keydown, and mouseover.

Event Object: The event object provides information about the event and the target element.

Removing Event Listeners: Use removeEventListener() to detach event handlers if needed.

Summary: Effective Event Management

23. JavaScript LocalStorage and Session Storage API

Overview: Web Storage APIs

The Web Storage API provides mechanisms for storing data on the client side. This tutorial covers localStorage and sessionStorage, which are used for persistent and session-based data storage respectively.

Example: Using LocalStorage and SessionStorage

We’ll demonstrate how to store and retrieve data using both localStorage and sessionStorage.

Code Example: Web Storage APIs

            
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Web Storage</title>
</head>
<body>
    <input type="text" id="localStorageInput" placeholder="Enter local storage data">
    <button id="saveLocalStorage">Save to LocalStorage</button>
    <button id="loadLocalStorage">Load from LocalStorage</button>

    <input type="text" id="sessionStorageInput" placeholder="Enter session storage data">
    <button id="saveSessionStorage">Save to SessionStorage</button>
    <button id="loadSessionStorage">Load from SessionStorage</button>

    <p id="output"></p>

    <script>
        // LocalStorage
        document.getElementById('saveLocalStorage').addEventListener('click', () => {
            const data = document.getElementById('localStorageInput').value;
            localStorage.setItem('localData', data);
        });

        document.getElementById('loadLocalStorage').addEventListener('click', () => {
            const data = localStorage.getItem('localData');
            document.getElementById('output').textContent = `LocalStorage data: ${data}`;
        });

        // SessionStorage
        document.getElementById('saveSessionStorage').addEventListener('click', () => {
            const data = document.getElementById('sessionStorageInput').value;
            sessionStorage.setItem('sessionData', data);
        });

        document.getElementById('loadSessionStorage').addEventListener('click', () => {
            const data = sessionStorage.getItem('sessionData');
            document.getElementById('output').textContent = `SessionStorage data: ${data}`;
        });
    </script>
</body>
</html>

Explanation: Web Storage Methods

localStorage: Stores data with no expiration time. Data persists across browser sessions.

sessionStorage: Stores data for the duration of the page session. Data is cleared when the page session ends.

Methods: Use setItem(), getItem(), and removeItem() to manage data.

Summary: Utilizing Web Storage

Web Storage APIs provide convenient ways to store data on the client side. Understanding the differences between localStorage and sessionStorage helps in choosing the right storage solution for your needs.

24. JavaScript Modules - Export/Import Syntax for ES6 Modules

Overview: Using JavaScript Modules

JavaScript modules enable you to break down your code into reusable components. This tutorial covers the export and import syntax introduced in ES6 for modularizing your code.

Example: Using Export and Import

We’ll create a module that exports functions and import them into another file.

Code Example: ES6 Modules

// math.js

                
    export function add(a, b) {
        return a + b;
    }

    export function subtract(a, b) {
        return a - b;
    }

// main.js

                
    import { add, subtract } from './math.js';

    console.log(add(5, 3)); // Outputs: 8
    console.log(subtract(5, 3)); // Outputs: 2

index.html

                
    <!DOCTYPE html>
    <html lang="en">
    <head>
        <meta charset="UTF-8">
        <meta name="viewport" content="width=device-width, initial-scale=1.0">
        <title>JavaScript Modules</title>
    </head>
    <body>
        <script type="module" src="main.js"></script>
    </body>
    </html>

Explanation: Module Syntax

export: Used to export functions, objects, or values from a module.

import: Used to import functions, objects, or values into another module.

Module Types: Ensure that your script is of type module in HTML or use a module bundler for Node.js environments.

Summary: Modular Programming

JavaScript modules facilitate modular programming by enabling code reuse and separation of concerns. By using the export and import syntax, you can create organized and maintainable codebases.
