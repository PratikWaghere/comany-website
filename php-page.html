    29. Building a Small Part of the App with Procedural PHP

    Create a Simple User Registration System

    Overview:

    Procedural PHP programming involves writing code in a sequential manner where functions and logic are executed in a specific order. This approach is useful for small applications or for learning PHP fundamentals.

    Example:

    Let's create a simple user registration system where users can sign up and view their profile.

    Code Example:

    
<?php
// Connect to the database
$mysqli = new mysqli("localhost", "user", "password", "database");

// Check connection
if ($mysqli->connect_error) {
    die("Connection failed: " . $mysqli->connect_error);
}

// User registration
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $username = $_POST['username'];
    $password = password_hash($_POST['password'], PASSWORD_BCRYPT);

    $stmt = $mysqli->prepare("INSERT INTO users (username, password) VALUES (?, ?)");

    1. Overview:

    2. Example:

    Implementing a transaction for transferring money between two accounts.

    3. Code Example:
    
<?php

    $pdo->beginTransaction();

    // Deduct from account A
    $stmt = $pdo->prepare("UPDATE accounts SET balance = balance - :amount WHERE id = :id");

    // Add to account B
    $stmt = $pdo->prepare("UPDATE accounts SET balance = balance + :amount WHERE id = :id");

    $pdo->commit();

    PHP WeakMap Explained

    1. Overview:

    PHP's WeakMap is a special type of map that holds weak references to objects, meaning the objects can be garbage collected when there are no strong references to them. This is useful for caching or tracking objects without preventing their cleanup.

    2. Example:

    You can use a WeakMap to cache objects without affecting their lifecycle, allowing PHP to garbage collect them when they are no longer needed.

    3. Code Example:
    
        class User {
            public $name;
            public function __construct($name) {
                $this->name = $name;
            }
        }
        
        $cache = new WeakMap();
        
        $user = new User('John');
        $cache[$user] = "Cached User Data";
        
        echo $cache[$user]; // Outputs "Cached User Data"
        
        // Unsetting the strong reference to $user
        unset($user);
        
        // Now $cache has no reference to the object
        var_dump($cache); // The WeakMap is empty because the user object was garbage collected
    4. Explanation:
    • WeakMap: A WeakMap holds weak references, meaning the referenced objects are not kept alive by the map. Once there are no strong references to the object, PHP garbage collects it.
    • Use Case: This is useful for caching, as it allows the application to track objects without preventing their automatic cleanup when no longer in use.
    5. Summary:

    WeakMap allows you to store objects without preventing their garbage collection, making it useful for memory-efficient caching or tracking of objects.
